üí° Key Insights

Understanding JobNimbus Data Architecture

  • Notes are embedded in Job objects - Not a separate resource with its own endpoint
  • Documents/Files use a unified endpoint - All attachments accessible via /api1/files
  • Web routes vs API endpoints - URLs without /api1/ prefix serve the web interface (HTML), not API data (JSON)
  • Client-side filtering is common - Many operations fetch larger datasets and filter in application code

‚ö†Ô∏è Why the Confusion?

  • ‚Ä¢ The web interface may display "Documents" or "Orders" as separate tabs/sections
  • ‚Ä¢ These are UI organizational concepts, not separate API endpoints
  • ‚Ä¢ The web frontend likely uses the same /api1/files and /api1/jobs endpoints we have access to
  • ‚Ä¢ Admin vs. API Key permissions are the same - both use the same API endpoints
