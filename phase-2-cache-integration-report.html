Integration Pattern

All tools follow a consistent integration pattern:

  1. Import Dependencies: withCache, CACHE_PREFIXES, getTTL
  2. Generate Cache Identifier: Deterministic key from all input parameters
  3. Wrap Execute Method: withCache() wrapper around core logic
  4. Specify Cache Strategy: Entity, operation, identifier, and TTL

Cache Key Format

Hierarchical cache key structure ensures uniqueness and organization:

jobnimbus:{entity}:{operation}:{identifier} Examples: jobnimbus:jobs:list:0:15:2025-10-01:2025-10-31:null:null:null:null:desc:compact jobnimbus:contacts:search:john:0:50:null:null jobnimbus:activities:list:0:15:2025-10-01:2025-10-31:null:null:null:null:desc

Graceful Degradation

The cache system includes robust error handling:

  • Circuit breaker pattern prevents cascade failures
  • Automatic fallback to direct API calls if cache unavailable
  • Error logging without disrupting user operations
  • Health monitoring at /cache/health
