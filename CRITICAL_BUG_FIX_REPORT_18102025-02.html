Issue #1: Corrupted Dates (1970-01-21)
PARTIALLY FIXED

Problem Description

All tasks were showing due_date = 1970-01-21 and days_overdue ‚âà 20,359. This occurred because the date_end field was null and converted to 0 milliseconds, interpreted as Unix epoch (1970-01-01).

Root Cause

  • API returns date_end = null for tasks without due dates
  • Normalization code only checked !task.date_end
  • Failed to catch date_end === 0 (epoch timestamp)

Fix Applied

Affected Files:
  • src/tools/analytics/getTaskManagementAnalytics.ts (line 582-585)
  • src/tools/analytics/getUserProductivityAnalytics.ts (line 599-603)
